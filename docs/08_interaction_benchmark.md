# 08. 相互作用モジュール Φ の比較ベンチマーク

「個人差の本体」である摂動×個人の相互作用 $\Phi$ を、**additive(=CPA相当) / FiLM /
cross-attention / hypernetwork** で比較する。さらに強力な単純ベースラインとして
**摂動ごとの線形 ridge**（$\Delta \sim [1, \text{genotype}]$）と population-mean を並べる。
スクリプト: `scripts/benchmark_interactions.py`。指標は `indiv_R2`（個人差成分の説明率, docs/04）。

## 1. 2 つのレジーム（合成データ）

- **線形**: $\text{gain}_{d,p} = 1 + a_p\cdot g_d$。個人差は genotype に線形 → 線形 ridge が上限に近い。
- **非線形**: $\text{gain}_{d,p}$ は飽和的 2 層 tanh MLP ＋ genotype 二次項。線形 ridge では
  捉えきれない → 表現力の高い $\Phi$ の出番。

## 2. 結果（leave-one-donor-out, train≈175 / test 25, 各150 epoch）

| Φ / baseline | 線形 indiv_R2 | 非線形 indiv_R2 |
|---|---:|---:|
| additive（CPA 相当, 相互作用なし） | −0.274 | −0.774 |
| FiLM | **+0.878** | −0.078 |
| cross-attention | +0.808 | **+0.182** |
| hypernetwork | **+0.981** | +0.062 |
| ridge（線形, 参照） | +1.000 | +0.187 |
| population-mean（対照） | −0.000 | −0.000 |

## 3. 読み取り（正直な結論）

1. **additive は両レジームで失敗**（負）。摂動効果を個人非依存に足すだけでは個人差を表現
   できない、という設計の核心仮説を裏づける。
2. **線形レジームでは ridge が最良**（1.0）。hypernet(0.98) と FiLM(0.88) が続く。
   → 個人差が単純（genotype に線形）なら、わざわざ深層を使う必要はない。
   **実データ評価では線形 ridge を必ずベースラインに入れる**こと。
3. **非線形レジームでは全手法が苦戦し ridge も 0.19 に崩落**。この設定（train 175）では
   cross-attention(0.18) が ridge と同等で最良、FiLM は伸びない。
   → 強い非線形個人差は **データ飢餓**で、モジュール選択以前にドナー数が効く（次節）。
4. **万能の Φ は無い**。線形では hypernet/ridge、非線形では cross-attn が有利で、
   レジーム（個人差の関数形）とデータ量に依存する。

## 4. データスケーリング（非線形レジーム）

非線形個人差はドナー数を増やすと学習可能になり、神経モジュールが ridge を **大きく** 超える
（test=40 の独立分割。少データ域はノイズが大きい点に注意）:

| train ドナー数 | FiLM | cross-attn | hypernet | ridge（線形） |
|---:|---:|---:|---:|---:|
| 200 | −0.194 | −0.203 | −0.069 | +0.070 |
| 500 | +0.709 | +0.392 | **+0.741** | +0.081 |
| 900 | +0.880 | +0.836 | **+0.900** | +0.156 |

- 少データ（200）では全手法が崩れ、ridge と大差ない（**データ飢餓が支配的**）。
- 中〜大データ（500→900）で神経モジュールが急伸し、**ridge(0.08→0.16 で頭打ち) を圧倒**。
  ridge は線形構造しか表現できないため、非線形個人差では原理的に伸びない。
- この域では **hypernet が最良**（0.90）、FiLM が僅差（0.88）、cross-attn が続く。

→ 「非線形な個人差を当てたいなら、表現力の高い $\Phi$ ＋ **十分なドナー数**」。
線形・少データなら ridge/FiLM で足りる。

## 5. 実務への含意

- まず **ridge と population-mean** をベースラインに固定する。
- 個人差が単純そう（線形・少データ）→ FiLM で十分。実装も安定。
- 個人差が複雑そう（非線形・多ドナー）→ cross-attention / hypernetwork を検討。
  ただしパラメータ増・データ要求に注意（hypernet は重みを生成するため最も要求が高い）。
- いずれも **leave-one-donor-out × indiv_R2** で「個人差を当てているか」を必ず検証する。
